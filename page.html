Figma token
A named design variable (e.g. color.background.primary) storing a value — color, spacing, border width. Updating a token propagates changes across every component that references it.
Design system
The collection of Figma tokens, components, and guidelines defining the shared visual language across all sites on the platform.
Theme
A named set of token values — Brand, Dark, Light, or White — applied to a full page or scoped to a single component to shift its visual style.
Brand
The identity layer applied on top of a theme. Each advisor may have their own brand (colors, fonts, logo) while sharing the underlying theme structure.
Component
A reusable UI building block (Billboard, MainNav, Cardlist…) built in Sitecore AI CMS, styled via Figma tokens, and rendered by the Next.js frontend.
Variant
An alternate version of a component with a different layout or behaviour. Variants share the token system but differ structurally.
Package
A curated bundle of components, variants, token configurations, and IA for a specific advisor website type. Each of the ~5 packages is a distinct, deployable site configuration.
Information architecture (IA)
The organisation of pages, navigation, and content hierarchy. Each advisor package can have its own IA independently of others.
Sitecore AI CMS
The content management system where components, variants, and page structures are configured. Figma tokens control the styling; authors manage content here.
Headless Next.js
The frontend application that fetches content from Sitecore and renders it. "Headless" means the CMS and presentation layer are decoupled.
Dotcom
The primary corporate website currently in development — a single-brand, unified experience built on the new system.
Advisor website
A site for individual financial advisors built using one of the ~5 packages. Each can have a distinct look, component set, and IA while sharing the core platform.
